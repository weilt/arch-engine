# Agent-Protocol-Toolkit (APT) 设计规格

**日期:** 2026-06-01  
**状态:** 已批准（superpowers 流程）  
**方案:** A — 规约最小实现

---

## 1. 背景与目标

APT 是一套全局工具集，用于强制规范 Claude Code 多代理协同开发流程。大模型在长任务中容易遗忘规则、伪造接口、逃避文档更新；APT 通过 **MCP Server（底层拦截）+ Shell 脚本（全局注入）+ Custom Commands（流程引导）+ TS 契约（编译校验）**，将软性 Prompt 规则转化为硬性工程约束。

**成功标准：**

- 在任何项目中运行 `agent-init` 后，子代理可通过 `/start-feature`、`/finish-feature` 被强制走契约查询/注册流程
- `query_contract` 找不到依赖时，子代理必须 `report_missing` 并停止
- `register_contract` 成功后自动更新 `.ai/INDEX.md`
- macOS 与 Windows 均可 install 并日常使用

## 2. 已确认决策

| 项 | 选择 |
|----|------|
| 源码位置 | 本仓库 `claude_plugin` |
| 安装目标 | `~/.apt/`（Windows: `%USERPROFILE%\.apt\`） |
| 跨平台 | `agent-init.sh` + `agent-init.ps1`；`install.sh` + `install.ps1` |
| 命令模板名 | `finish-feature.md`（非 `end-feature.md`） |
| MCP 配置 | 全局 `~/.claude/settings.json`，install 自动合并 |
| 数据隔离 | MCP Server 读 `process.cwd()/.ai/db.json` |
| 安装方式 | 纯 Shell/PowerShell，复制 build 产物到 `~/.apt/` |
| 实现深度 | v1 仅文件存在性校验，不解析 TS AST |

```
~/.apt/

├── bin/
```

### 3.4 运行时流程

1. **一次性 install**：构建 MCP → 复制到 `~/.apt/` → 合并 MCP 到 `~/.claude/settings.json` → 追加 PATH
2. **每项目 agent-init**：注入 commands → 初始化 `.ai/db.json`
3. **开发**：`/start-feature` → `query_contract` / `report_missing`；`/finish-feature` → `register_contract`

## 4. 数据模型

### 4.1 `.ai/db.json`

路径：`<project-root>/.ai/db.json`（MCP Server 以 `process.cwd()` 为项目根）

```json
{
  "contracts": [],
  "missingRequests": []
}
```

**Contract 记录：**

```typescript
interface Contract {
  name: string;           // 唯一，如 "UserService.getProfile"
  description: string;
  tsFilePath: string;     // 相对项目根或绝对路径
  registeredAt: string;   // ISO 8601
}
```

**MissingRequest 记录：**

```typescript
interface MissingRequest {
  missingName: string;
  reason: string;
  reportedAt: string;     // ISO 8601
}
```

**规则：**

- `register_contract`：`name` 已存在则拒绝覆盖
- `tsFilePath` 解析：相对路径相对于 `process.cwd()`；绝对路径直接使用
- 若 `.ai/` 或 `db.json` 不存在，MCP Tool 返回明确错误，提示先运行 `agent-init`

### 4.2 `.ai/INDEX.md`

由 `register_contract` 成功后自动全量 regenerate（非增量）。

```markdown
# Agent Protocol Contract Index

> Auto-generated by agent-protocol-mcp. Do not edit manually.

## Registered Contracts

| Name | Description | TS File | Registered |
|------|-------------|---------|------------|
| UserService.getProfile | 获取用户资料 | src/contracts/user.ts | 2026-06-01T12:00:00Z |

## Pending Missing Requests

| Name | Reason | Reported |
|------|--------|----------|
```

若无记录，对应章节写 `_None._`

## 5. MCP Server

**技术栈：** Node.js ≥18、TypeScript、`@modelcontextprotocol/sdk`  
**协议：** stdio  
**项目根：** `process.cwd()`

### 5.1 `query_contract`

| 参数 | 类型 | 必填 |
|------|------|------|
| `name` | string | 是 |

**逻辑：**

1. 读取 `.ai/db.json`；文件/目录缺失 → 错误 `"❌ .ai/db.json not found. Run agent-init first."`
2. 在 `contracts` 中按 `name` 精确匹配
3. **存在：** 返回 `{ description, tsFilePath, tsContent }`；`tsContent` 为 TS 文件全文（读取失败则 `tsContent: null` + 警告）
4. **不存在：** 返回 `"❌ Contract not found. You must call report_missing to block this task."`

### 5.2 `register_contract`

| 参数 | 类型 | 必填 |
|------|------|------|
| `name` | string | 是 |
| `description` | string | 是 |
| `tsFilePath` | string | 是 |

**逻辑：**

1. 解析 `tsFilePath` 为绝对路径，检查文件存在；不存在 → `"❌ TS file not found. Registration aborted."`
2. 读取 `db.json`；`name` 已存在 → `"❌ Contract '{name}' already exists. Registration aborted."`
3. 追加到 `contracts`，写入 `registeredAt`
4. 调用 `regenerateIndexMd()`
5. 返回 `"✅ Contract registered and INDEX.md updated."`

### 5.3 `report_missing`

| 参数 | 类型 | 必填 |
|------|------|------|
| `missingName` | string | 是 |
| `reason` | string | 是 |

**逻辑：**

1. 追加到 `missingRequests`，写入 `reportedAt`
2. regenerate `INDEX.md`（含 missing 段）
3. 返回 `"🚫 TASK BLOCKED. Missing dependency reported to Manager Agent. Stop current work immediately and wait for resolution."`

## 6. agent-init

**Unix (`bin/agent-init.sh`)：**

1. 创建 `.claude/commands/`（若不存在）
2. 复制 `$APT_HOME/templates/*.md` → `.claude/commands/`（`APT_HOME` 默认 `~/.apt`，可通过环境变量覆盖）
3. 创建 `.ai/`，若 `db.json` 不存在则写入空结构
4. 输出 `"✅ Agent Protocol Toolkit initialized. Commands injected."`

**Windows (`bin/agent-init.ps1`)：** 同上，路径用 `$env:USERPROFILE\.apt` 或 `$env:APT_HOME`

**不修改** 项目级 `settings.json`（MCP 已在全局配置）。

## 7. install 脚本

### 7.1 共同步骤

1. 检测 Node.js ≥18
2. 在仓库 `mcp-server/` 执行 `npm ci && npm run build`
3. 创建 `~/.apt/` 目录结构
4. 复制：`dist/`、`package.json` → `~/.apt/mcp-server/`；在目标目录 `npm ci --omit=dev`
5. 复制 `templates/`、`bin/`、`README.md`
6. 合并 MCP 配置到 `~/.claude/settings.json`（见 7.2）
7. 追加 PATH（见 7.3）
8. 输出安装摘要与验证命令

### 7.2 MCP settings 合并

目标文件：

- Unix: `~/.claude/settings.json`
- Windows: `%USERPROFILE%\.claude\settings.json`

写入/合并：

```json
{
  "mcpServers": {
    "agent-protocol-mcp": {
      "command": "node",
      "args": ["<ABSOLUTE_PATH_TO>/.apt/mcp-server/dist/index.js"]
    }
  }
}
```

- 若文件不存在，创建含上述内容的新 JSON
- 若已存在，深度合并 `mcpServers.agent-protocol-mcp`（覆盖同名校验）
- 使用绝对路径，Windows 用反斜杠或 JSON 转义正确

### 7.3 PATH

- **Unix：** 在 `~/.bashrc` 或 `~/.zshrc` 追加 `export PATH="$HOME/.apt/bin:$PATH"`（若尚未存在）
- **Windows：** 用户 PATH 环境变量追加 `%USERPROFILE%\.apt\bin`；提供 `agent-init.cmd` 包装调用 ps1

## 8. 命令模板

**权威源：** `templates/`（`ref-docs/` 保留参考，内容同步）

与规约一致：规划依赖 → 强制 `query_contract` → 未找到则 `report_missing` 并停止 → 输出计划等确认。

## 9. 错误处理

| 场景 | 行为 |
|------|------|
| 未 agent-init | MCP 返回明确错误，不创建文件 |
| db.json 损坏 | 返回 JSON parse 错误，不写入 |
| 并发写入 | v1 不做文件锁；单代理场景可接受 |
| install 无 Node | 中止并提示安装 Node 18+ |
| settings.json 非法 JSON | 备份为 `.bak` 后重建或报错退出 |
