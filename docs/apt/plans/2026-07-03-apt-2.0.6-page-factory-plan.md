# APT 2.0.6 页面工厂 Implementation Plan

> **Spec:** `docs/superpowers/specs/2026-07-03-apt-2.0.6-page-factory-design.md`
> **Command:** `/plan-from-spec`
> **Status:** approved

**Goal:** 补齐「开发逐页冻结 logic → 全页 approved 门禁 → rollup spec + plan 批量实现」的页面工厂能力，使 2.0.6 成为单语言栈下的完整开发闭环（3.0 多语言不在范围内）。

**Architecture:** 纯模板/Skill/脚本增量，不新增 MCP 工具与斜杠命令。新增 `apt-v0-handoff` Skill（`EXTRA_SKILLS` 分发）、`_pages.md` 进度板、`check-v0-freeze` 门禁、rollup spec 骨架；扩展 `plan-from-spec` / `feature` 模板固化 Phase A 门禁与 B1/B2/B3 单页 Task 结构。依赖已落地的 v0 `design-sync --adapter v0`（若工作区未提交则 Task 0 先收口）。

---

## Part 1 — 技术方案（APT 寻址）

### 1.1 范围与约束

**范围内：**

- `apt-v0-handoff` Skill：开发对照 v0 + PM 文档逐页写 manifest/logic，更新 `_pages.md`
- `designs/v0/_pages.md` 模板与狗食示例
- `scripts/check-v0-freeze.mjs`（或等价）Phase A 硬门禁
- `templates/page-rollout-spec.md` rollup spec 骨架
- `templates/plan-from-spec.md`：门禁检查 + 单页 B1/B2/B3 Task 指引
- `templates/feature.md`：logic 冻结 SSOT、全页批量须 rollup plan
- `inject-platform-assets.cjs` → `EXTRA_SKILLS` 分发 `apt-v0-handoff`
- `templates/_agents-md-snippet.md` / README 两阶段工厂说明
- 修订 `templates/design-page.md` 附录 A：主路径指向 `apt-v0-handoff`（PM 不写双文件）
- 狗食：`v0-fixture` + `_pages.md` + freeze 脚本 + inject 测试

**非目标（spec §1.2 / §10）：**

- 3.0 多语言 / 多栈
- 新 MCP 工具、新斜杠命令（`PUBLIC_TEMPLATES` 仍为 10）
- `apt-goal` 自动读 `_pages.md` 排 Task（2.1）
- 修改 `loopDone` / `aggregate.ts` 状态机（Phase A 仅文档约定 `page_handoff`）
- verify 自动 diff logic 与实现

**前置：** `2026-07-02-v0-page-handoff` 引擎层（`ingest/v0.ts`、`design-sync --adapter v0`、`query_design` logicMarkdown/gaps）须已合并或通过 Task 0 收口。

### 1.2 设计寻址

本 plan 为 APT 插件工作流，**无插件自身 UI**。狗食与业务页约束引用设计层：

| 项 | MCP 结果 | 约束摘要 |
|----|----------|----------|
| 全局 tokens/style/bindings | `query_design(scope: global)` ✅ | `designs/apt-reference-ds`；vue + element-plus；handoff 时 `query_design(global)` 列语义组件 |
| 狗食页 user-list | `query_design(page: user-list)` ❌（未 sync 到本仓 `.ai/design/`） | 狗食测试在临时目录 `design-sync --adapter v0` 后验证；非阻塞本 plan |
| 参考 list/form | `query_design(page: list-page)` ✅（实现时抽检） | pageType 软对齐 |
| PageHeader / EmptyState | 实现时 `query_design(component:)` | handoff logic §依赖 用真实 id |
