# APT 2.0 自主闭环（Solo Autonomous Loop）Implementation Plan

> **Spec:** `docs/superpowers/specs/2026-06-25-apt-2.0-autonomous-loop-design.md`
> **Command:** `/plan-from-spec`
> **Status:** approved

**Goal:** 为 APT 增加「单人自主交付完整产品」的闭环能力——`/apt-goal` 外层编排、`query_project_status` 状态中心、`/auto-brainstorm` 风险分级、`loopDone` 硬条件，使产品目标可跨会话推进直至 verify PASS 交付。

**Architecture:** 在 `mcp-server` 新增第 16 个只读 MCP 工具 `query_project_status`，由纯函数 `status/` 模块聚合 `.apt/` 状态文件与 arch verify 结果，重算 phase / loopDone / nextAction。新增 3 个 `/` 命令（`apt-goal`、`auto-brainstorm`、`current-status`）+ `apt-status` CLI + hook 示例，沿用「模板 SSOT + `inject-platform-assets.cjs` 分发」模式。`agent-init` 初始化 `.apt/` 状态文件。
